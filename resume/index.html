  • 项目背景：供应链系统为了能够更好的精细化的去管理商品库存。

  • 担任角色：负责供应链系统的技术架构、核心代码开发。

  • 项目功能：供应链系统分为ERP和WMS。主要功能有：公司、商品、供应商、采购、指导作业、入库、上架、分拣、出库、盘点、调拨、库位、发运 ...等等功能。

  • 开发组件：PHP7、Mysql5.6、Nginx、Laravel5.5、Docker、Jenkins、Redis。

  • 我的业绩：

    • 服务设计方面采用独立、减少依赖思想。供应链分为ERP、WMS、FILE、USER、STORAGE等子系统。

    • 为保障服务具有更高的稳定性，避免单点故障，服务都是采用多节点，配套完善的日志系统、报 警系统。

    • 服务应用、数据依赖、解耦方面，采用RabbitMq和数据中心解决。

    • 主导构建完善的任务管理系统、WIKI、API文档、监控等系统。

    • 主导设计、架构、代码规范、代码优化、核心代码开发。

    • MYSQL采用主从架构，搜索引擎使用ES、用户权限数据使用Redis存储。

    • 使用 HTTP 1.1 流水线的持久连接，增加复用性。

    • 提升API性能上，用Guzzlehttp代替Curl。

    • 防止CSRF、XSS攻击方面，使用令牌和数据的严格验证来做验证。

    • 为了解决代码复用性、数据多入口和出口、人员协同、边界等问题，采用了Service层设计，面 向Interface调用和编码。

    • 采用 Trait、Collection 增加基础类代码的复用性

    • 在代码扩展性和易用性上，增加了单一职责类、状态类、外观模式、链式调用等。

    • 采用Tideways监控API系统、异常钉钉通知。

    • 核心数据，增加数据快照，数据巡查。

# 系统优化

北京果壳宇宙教育科技有限公司 2017.1 - 2017.2

  • 项目背景：做春节活动优化系统

  • 担任角色：技术组长

  • 我的业绩：

    • 整站优化思路：不要单点、流量限制、请求路径缩短、请求数减少、减少依赖、发现热 点、数据隔离、业务隔离

    • 数据隔离、业务隔离方面，活动有单独的集群服务和数据库。避免因为活动拖垮这个服 务。

    • 流量限制使用Nginx limit_rate 和 limit_rate_after模块。

    • 发现热点数据，动静分离。热点数据直接使用热点缓存，减少请求路径。

    • CSS、Js、Img 存储CDN，多个css和Js合并压缩。减少请求数。

    • 优化MYSQL参数：连接数、索引块的缓冲区大小、缓存数据块和索引键、事务日志所 使用的缓冲区、缓存结果集等参数。

    • 优化PHP参数，如：PHP-Fpm中pm进程数和内存比例、运行模式为dynamic。PHP大 页内存、开启Opcache、超时时间等。

    • 优化Nginx参数，如：超时时间、进程数、打开文件数、epoll的IO模型、连接数、缓冲 区大小。

    • WEB服务采用集群，负载均衡方案选用当前连接数最少的server来处理
