# 502

Bad Gateway，网关错误

上游无响应 上游超时

502并不是指网关本身出了问题，而是从上游接收响应出了问题，比如由于上游服务自身超时导致不能产生响应数据，或者上游不按照协议约定来返回数据导致网关不能正常解析。

# 案例1

  • 关闭PHP-FPM进程

# 案例2

  • 脚本执行时间，超过PHP-FPM允许的超时时间
    sleep(10)
    1
    request_terminate_timeout = 5
    1

# 504

Gateway Timeout，网关超时

上游无按规定时间返回响应

504是指在php-fpm还未执行完成的某一时刻，由于超过了nginx自身的超时时间，nginx则以为上游php-fpm没有按照设置时间返回响应数据就会返回504， 此时对于php-fpm而言还会继续执行下去，直到执行完成。

# 案例

  • php代码
sleep(10)
1
  • php-fpm
request_terminate_timeout=30
1
  • nginx
fastcgi_read_timeout 5;
1
