# HTTP 1.0

描述

HyperText Transport Protocol）是超文本传输协议的缩写。

HTTP协议采用了请求/响应模型。

HTTP协议是服务器传输超文本到本地浏览器的传送协议

HTTP协议信息是明文传输

HTTP协议在应用层

  • 请求报文和相应报文参数

# 主要特点

# 简单快速

客户向服务器请求服务时，只需传送请求方法和路径。请求方法常用的有GET、HEAD、POST。每种方法规定了客户与服务器联系的类型不同。

# 灵活

HTTP允许传输任意类型的数据对象。正在传输的类型由Content-Type加以标记。

# 无连接

无连接的含义是限制每次连接只处理一个请求。服务器处理完客户的请求，并收到客户的应答后，即断开连接。采用这种方式可以节省传输时间。

# 无状态

HTTP协议是无状态协议。无状态是指协议对于事务处理没有记忆能力。缺少状态意味着如果后续处理需要前面的信息，则它必须重传，这样可能导致每次连接传送的数据量增大。另一方面，在服务器不需要先前信息时它的应答就较快。

也就是说，同一个客户端第二次访问同一个服务器上的页面时，服务器无法知道这个客户端曾经访问过，服务器也无法分辨不同的客户端。

# 请求响应报文

响应报文组成：

  • 状态行
  • 响应头部
  • 响应正文

# 状态行

  • 200-299 表示成功
  • 300-399 指资源重定向
  • 400-499 客户端请求出错
  • 500-599 服务端出错

常用状态：

Status Desc
200 请求成功
302 请求成功
400 客户端请求有语法错误，不能被服务器识别
403 服务器接收到请求，但是拒绝提供服务（认证失败）
404 请求资源不存在
500 服务器内部错误
502 错误网关
503 服务不可用,表示服务器当前不能够处理客户端的请求，在一段时间之后，服务器可能会恢复正
504 网关超时

# 响应头部

  • 常见响应头部
Key Value Desc
Connection keep-alive
close		 连接方式
Content-Type application/json; charset=utf-8
Date Tue, 26 Feb 2019 03:15:51 GMT
Server Nginx 服务器用来处理请求的软件信息及其版本
它和 User-Agent 请求报头域是相对应的，前者发送服务器端软件的信息，后者发送客户端软件(浏览器)和操作系统的信息。
  • Connection

    • keep-alive

      在完成本次请求的响应后，保持连接，等待本次连接的后续请求

    • close

      在完成本次请求的响应后，断开连接，不等待本次连接的后续请求了
